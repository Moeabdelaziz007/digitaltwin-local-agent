# 🤖 Autonomous Venture Lab Agents (MAS-ZERO)

This document defines the specialized agent desk and the **Autonomous Venture Engine (AVE)** hierarchy utilized by the Venture Lab to identify, validate, and execute profit-focused opportunities.

---

## 🏗️ The Command Hierarchy

### 1. Consensus CEO (The Orchestrator)
- **Role**: Final verification of the MAS cycle and manager of the **Venture Engine (AVE)**.
- **Responsibility**: Manages the "Experience Library" and approves autonomous execution tickets.
- **2026 Logic**: Orchestrates the "6 Revenue Rivers" sub-teams.

### 2. Meta Architect (The Designer)
- **Role**: Designs the optimal DAG (Directed Acyclic Graph) for complex tasks.
- **Capabilities**: Dynamically rewrites agent instructions based on venture-specific missions.
- **Inference Pattern**: Generates strict JSON configurations for specialized workers.

### 3. Opportunity Scout (The Sensor)
- **Role**: Real-time sensing across market signals (Upwork, GitHub, Reddit, IndieHackers).
- **Core Loop**: `Scan` → `Refract` (Jina) → `Identify`.
- **Output**: Alpha signals with "Profit Probability" metadata.

### 4. Risk Manager (The Protector)
- **Role**: Ensures capital safety and legal defensibility.
- **Veto Power**: Automatically pauses any autonomous action that exceeds the venture's token budget.

### 5. Execution Specialist (The Finisher)
- **Role**: Atomic task execution using the **PTY Bridge**.
- **Interface**: Direct control over local tools, browser automation, and GitHub CLI.

## 📐 Logic: The Self-Execution Loop
Agents do not just "talk"—they **commit**. The new loop follows:
1.  **Sense**: Scout identifies an opportunity.
2.  **Think**: CEO & Architect draft a solution DAG.
3.  **Execute**: Specialist performs the action via PTY.
4.  **Audit**: Every action is logged as an immutable **Ticket**.
5.  **Learn**: Memory Engine summarizes the outcome for the next cycle.
